## 二、认证拦截机制

### 2.1 AuthFilter.java - 登录拦截器

**工作原理：**
```
用户请求 → Servlet容器 → 匹配URL模式 → 执行doFilter() → 检查Session
```

**关键代码：**
```java
@WebFilter(urlPatterns = {"/profile.jsp", "/education", "/education/*", "/photo/*"})
public class AuthFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) {
        HttpSession session = httpReq.getSession(false);
        if (session != null && session.getAttribute("user") != null) {
            chain.doFilter(request, response);  // 已登录，放行
        } else {
            httpResp.sendRedirect("/login");     // 未登录，重定向
        }
    }
}
```

**拦截规则：**
| URL 模式 | 匹配示例 |
|----------|----------|
| `/profile.jsp` | `/profile.jsp` |
| `/education` | `/education` |
| `/education/*` | `/education/add`, `/education/edit` |
| `/photo/*` | `/photo/upload`, `/photo/list` |

**无需拦截的页面：**
- `/login`, `/register` - 登录注册
- `/index.jsp` - 首页
- `/css/*`, `/js/*` - 静态资源

**特点：**
- 使用 `@WebFilter` 注解，无需在 `web.xml` 中配置
- Servlet 容器启动时自动扫描注册
- 对业务代码零侵入

```java

    // 已登录

// 方式2：直接获取用户对象
User user = (User) req.getSession().getAttribute("user");
if (user == null) {
    // 未登录

```

- 每次请求创建新连接
- 适合小型项目

### 6.2 DAO 模式示例

```java
        String sql = "INSERT INTO photos (...) VALUES (?, ?, ...)";

        }
    }
    
    public List<Photo> findAll() {
        String sql = "SELECT * FROM photos ORDER BY upload_time DESC";
        // 执行查询，映射到对象列表
    }
}
```

**使用 PreparedStatement 防止 SQL 注入**

## 八、请求转发与重定向

### 8.1 转发 (Forward)

```java
// 服务器内部转发，URL不变
req.getRequestDispatcher("/login.jsp").forward(req, resp);
```

**特点：**
- 可以共享 request 属性
- 浏览器地址栏不变
- 只能转发到本应用内的资源

### 8.2 重定向 (Redirect)

```java
// 浏览器重新请求，URL改变
resp.sendRedirect(req.getContextPath() + "/login");
```

**特点：**
- 浏览器地址栏改变
- 不能共享 request 属性
- 可以重定向到任意 URL

### 8.3 使用场景

| 场景 | 使用方式 |
|------|----------|
| 登录成功后跳转 | 重定向 (防止刷新重复提交) |
| 表单验证失败 | 转发 (保留错误信息) |
| 登录后访问受保护页面 | 重定向到登录页 |
